JavaScript中的this

2016.04.27 Justin Wong

JavaScript中的this是执行上下文（execution context, EC）的一个属性，而不是变量对象（variable object, VO）的一个属性。

this和上下文的可执行代码类型直接相关。this的值在进入上下文阶段就确定了，并且在执行代码阶段不能改变。

全局代码中this的值

在全局代码中，this的值始终是全局对象本身。

函数代码中this的值

函数代码中this的值最主要的特性是：它不是静态绑定在函数上的。
this的值是在进入上下文阶段确定的，对于同一函数代码，进入不同的函数上下文，this的值可能不一样。
一旦到了执行代码阶段，this的值就不能改变，比如不能给this赋新值。

在一般的函数调用中，this是由激活上下文代码的调用者提供的，比如调用函数的外层上下文，this的值取决于调用函数的方式，this的值随着函数调用方式的不同而不同。

在这种情况下，with和catch是例外的。

在with语句中调用函数，并且with对象包含一个函数属性时，with语句会将该对象添加到作用域链的最前端，在活跃对象之前。调用括号左侧为引用类型的值时，base对象不再是活跃对象，而是with语句的对象。无论是内存函数还是全局函数都如此，因为with对象掩盖了作用域中更高层的对象（全局对象或活跃对象）。

当调用的函数是catch从句的参数时也是这样。在ECMA-262-3中，catch对象会添加到作用域链的最前端，this的值设置为catch对象。但在ECMA-262-5中，this的值设置为全局对象。

递归调用一个非匿名函数也会发生这种情况。在第一次函数调用时，base对象是外层的活跃对象（或全局对象），在下一次递归调用时，base对象应该是存储着函数表达式可选名称的特定对象，但在这种情况下，this的值总被设置为global。
