JavaScript中的执行上下文和作用域

2016.04.26 Justin Wong

这篇博文主要参考 Dmitry Soshnikov 的几篇关于ECMA-262-3的文章，包含内容有：

  • 执行上下文栈（execution context stack, EC Stack）
  • 执行上下文（execution context, EC）
  • 变量对象（variable object, VO）和活跃对象（activation object, AO）
  • 作用域链（scope chain）

注意：本文描述的是ECMA-262-3。

执行上下文栈

ECMAScript中有三种不同类型的代码：全局代码、函数代码和eval代码，所有代码都是在其执行上下文中被求值。全局执行上下文只有一个，而函数执行上下文和eval执行上下文可能有多个。

每调用一个函数/eval函数，就会创建并进入一个具有新状态的执行上下文。所以一个函数可能会产生多个执行上下文。

一个函数可能会调用另一个函数，即一个执行上下文会触发另一个执行上下文。为实现执行上下文间的切换，需要一个执行上下文栈（EC Stack）来支持。

一个触发其他上下文的上下文叫caller，被触发的上下文叫callee。当caller触发/调用了callee，caller会停止自身的执行并将控制流交给callee，callee自身会被压入执行上下文栈中并成为运行中/活跃的执行上下文。callee上下文结束后，出栈并将控制权返回给caller，那么当前运行中的上下文就变成caller上下文，caller上下文继续执行。

执行上下文

一个执行上下文（EC）可以抽象成一个简单的对象。每个执行上下文都由以下属性组成：变量对象（VO）、作用域链（scope chain）、this值（可能还有其他属性）。

可执行代码类型和执行上下文相关。可执行代码有三种类型：全局代码、函数代码和eval代码。

变量对象和活跃对象

变量是和执行上下文密切相关的，那么执行上下文就该知道数据存储在哪里以及如何获取数据，这种机制就称为变量对象（VO）。

变量对象存储以下内容：

  • 变量声明（VariableDeclaration，var）
  • 函数声明（FunctionDeclaration，FD）
  • 函数形参（function formal parameters）

全局对象：是一个在进入任何执行上下文之前就创建出来的对象。该对象以单例的形式存在。它的属性在程序的任何地方都能直接访问，其生命周期随着程序的结束而终止。

在创建全局对象时，它的一些属性如Math、String、Date、parseInt等也会被初始化，一些对象也会指向全局对象本身：如在BOM中，window对象指向全局对象。

引用全局对象的属性时通常省略前缀。

全局上下文中的变量对象就是全局对象。所以在全局上下文中声明一个变量时，可以通过全局对象上的属性来间接引用该变量。

函数上下文中的变量对象：

在函数执行上下文中，变量对象（VO）是不能直接访问的，它扮演活跃对象（VO）的角色。

在进入函数上下文时，活跃对象被创建，初始化时会创建一个arguments属性，它的值是arguments对象。

arguments对象包含了以下属性：

  • callee：对当前函数的引用
  • length：实参的个数
  • properties-indexes：其值是函数参数的值，properties-indexes的个数 == arguments.length。

处理上下文代码的过程

执行上下文代码的处理过程分为两步：

  • 进入执行上下文
  • 执行代码

进入执行上下文阶段：

进入执行上下文但没有执行代码时，活跃对象（VO）就会被以下属性填充（按顺序）：

  • 函数形参：其属性名即形参名，其值即实参值；对于没有传递的参数，其值为undefined
  • 函数声明：其属性名和值都是函数对象创建的；如果变量对象已包含同名属性，则替换原来的值
  • 变量声明：其属性名即变量名，其值为undefined；如果变量对象和已经声明的函数名或函数参数名相同，则不会影响已存在的属性

函数的生命周期：

函数的生命周期分为创建阶段和调用阶段。

创建阶段：
[[Scope]]是一个包含所有上层变量对象的分层链，它属于当前的函数执行上下文，并在函数创建阶段保存在函数中。
[[Scope]]是在函数创建的时候就保存起来、静态不变的，只创建一次后，一直存在直至函数销毁。
[[Scope]]是函数的属性，而Scope是执行上下文的属性。

调用阶段：
调用函数就会进入函数执行上下文，这时会创建活跃对象并确定this的值以及作用域链。
