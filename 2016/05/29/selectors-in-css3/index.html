CSS3选择器

2016.05.29 Justin Wong

关注CSS规范、浏览器支持和兼容性的问题：
CSS reference - MDN
CSS参考手册
STANDARDS AND DRAFTS
Can I use

目前CSS3规范中选择器的完整列表见：Selectors Level 3

CSS3中的选择器主要分为以下几类（包括CSS1和CSS2）：

  • 基本选择器
  • 层次选择器
  • 伪类选择器
  • 伪元素选择器
  • 属性选择器

基本选择器

选择器 名称 描述
* 通配选择器 选择所有HTML元素
E 元素选择器 选择指定类型的HTML元素
#id ID选择器 选择ID属性值为“id”的任意类型元素
.class 类选择器 选择class属性值为“class”的任意类型的任意多个元素
selector1,selectorN 群组选择器 将每一个选择器匹配的元素集合并

层次选择器

选择器 名称 描述
E F 后代选择器/包含选择器 选择所有被元素E包含的元素F
E>F 子选择器 选择所有作为元素E的子元素F
E+F 相邻选择器 选择紧贴在元素E后的元素F
E~F 兄弟选择器 选择元素E的所有位于E之后的兄弟元素F

注意：

  • 后代选择器和子选择器的区别在于：后代选择器选择的是所有的后代，包括子元素、孙子元素等等，而子选择器选择的仅仅只有子元素。
  • 相邻选择器和兄弟选择器的区别在于：相邻选择器选择的是紧邻的后面那个并且符合条件的元素，而兄弟选择器选择的是所有符合条件的并且位于元素E之后的同级元素。

伪类选择器

伪类选择器可以细分为：

  • 动态伪类选择器
  • 目标伪类选择器
  • 语言伪类选择器
  • UI元素状态伪类选择器
  • 结构伪类选择器
  • 否定伪类选择器

动态伪类选择器

选择器 描述
E:link 设置超链接a在未被访问前的样式
E:visited 设置超链接a在其链接地址已被访问过的样式
E:hover 设置元素在其鼠标悬停时的样式
E:active 设置元素被用户激活时（在鼠标点击与释放之间的事件）的样式
E:focus 设置元素获得焦点时（元素的onfocus事件发生）的样式

注意：

  • 超链接伪类的书写顺序必须遵循“爱恨原则”（LoVe/HAte），即需要按照“link-visited-hover-active”的顺序，这样的顺序是最可靠的。
  • E:link和E:visited只对a标签有效。
  • webkit内核浏览器会默认给:focus状态的元素加上outline的样式。

目标伪类选择器即E:target，作用是选择匹配E的所有元素，并且匹配元素被相关URL指向。这里可能不好理解，需要用一个例子来解释。

UI元素状态伪类选择器

选择器 描述
E:checked 匹配用户界面上处于选中状态的元素E
E:enabled 匹配用户界面上处于可用状态的元素E
E:disabled 匹配用户界面上处于禁用状态的元素E

注意：

  • E:checked用于type为radio或checkbox的input元素，以及select元素中的option元素。

结构伪类选择器

选择器 描述
E:root 匹配文档的根元素
E:first-child 匹配父元素的第一个子元素E
E:last-child 匹配父元素的最后一个子元素E
E:only-child 父元素只包含一个子元素，且该子元素匹配元素E
E:nth-child(n) 匹配父元素的第n个子元素E
E:nth-last-child(n) 匹配父元素的倒数第n个子元素E
E:first-of-type 匹配同类型中的第一个同级兄弟元素E
E:last-of-type 匹配同类型中的最后一个同级兄弟元素E
E:only-of-type 父元素只包含一个同类型的子元素，且该子元素匹配元素E
E:nth-of-type(n) 匹配同类型中的第n个同级兄弟元素E
E:nth-last-of-type(n) 匹配同类型中的倒数第n个同级兄弟元素E
E:empty 匹配没有任何子元素（包括文本节点）的元素E

对于E:root，在HTML文档中，根元素始终是html。
Demo

E:first-child和E:first-of-type等这两类选择器的区别在于：对于E:first-child要求元素E必须是其父元素的第一个子元素才能匹配，而E:first-of-type不需要。也就是说E:first-child会考虑父元素中的所有子元素，而E:first-of-type则考虑元素类型。其他类似的选择器也是这样的。
Demo

对于E:nth-child(n)、E:nth-last-child(n)、E:nth-of-type(n)、E:nth-last-of-type(n)中的n，表示的是第几个或者倒数第几个。
n有以下几种类型的取值：

  • 常数，如E:nth-child(0)：第0个元素，不存在、E:nth-child(3)：第3个元素
  • 关于n的表达式，需要注意的是，n的取值从0开始，取0,1,2,3,…下面是一些常见的例子：
    • E:nth-child(n+3)：匹配第3个及后面的所有元素E
    • E:nth-child(-n+3)：匹配第1~3个元素E
    • E:nth-child(2n+1)：匹配奇数位的元素E
    • E:nth-child(2n)：匹配偶数位的元素E
  • 奇偶关键字：odd/even：
    • E:nth-child(odd)：同E:nth-child(2n+1)
    • E:nth-child(even)：同E:nth-child(2n)

伪元素选择器

选择器 描述
E:first-letter / E::first-letter 匹配文本块的第一个字符
E:first-line / E::first-line 匹配元素的第一行文本
E:before / E::before 设置在元素前的内容，结合content属性使用
E:after / E::after 设置在元素后的内容，结合content属性使用
E::placeholder 设置元素文字占位符的样式
E::selection 设置元素被选择时的样式

注意：

  • 对于前四个伪元素选择器，两种写法都是可以的，单冒号是CSS2的写法但现在也支持，双冒号是CSS3的写法，如果不需要兼容老版浏览器，推荐使用双冒号表示伪元素选择器。
  • 对于E::first-letter和E::first-line，元素E的display属性的值必须是block、inline-block、table-cell、list-item或table-caption（块级元素），否则没有效果。

  • 对于E::before和E::after，必须定义content属性。
    content的取值如下：

  • E::before会影响到E::first-letter和E::first-line的选择。

  • 目前E::placeholder不是CSS3规范，需要添加特定浏览器的前缀：Demo
  • E::selection用于设置元素被选择时的样式，也不是CSS3规范（在CSS3草案中，规范中被移除），一般可以设置color、background-color、text-shadow（IE11不支持）：Demo

属性选择器

选择器 描述
E[attr] 选择具有属性attr的元素E
E[attr=”val”] 选择具有属性attr且属性值等于val的元素E
E[attr~=”val”] 选择具有属性attr且属性值具有多个以空格分隔的值、其中一个等于val的元素E
E[attr^=”val”] 选择具有属性attr且属性值是以val开头的字符串的元素E
E[attr$=”val”] 选择具有属性attr且属性值是以val结尾的字符串的元素E
E[attr*=”val”] 选择具有属性attr且属性值是包含val的字符串的元素E
E[attrl=”val”] 选择具有属性attr且属性值等于val或者是以val-开头的字符串的元素E

注意：

  • 最后一个是E[attr|=”val”]（因为不知道怎么转义，只好用l代替…）
