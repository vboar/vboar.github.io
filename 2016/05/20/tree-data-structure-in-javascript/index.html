JavaScript数据结构——树

2016.05.20 Justin Wong

树是一种非顺序的数据结构，也是一种分层抽象的数据模型，它对于存储需要快速查找的数据非常有用。

树的基本概念

下面将围绕上图来理解树的基本概念。

一个树结构包含一系列存在父子关系的节点。节点即树中的元素。
位于树顶部的节点叫根节点。图中节点1为根节点。
除了根节点外，树中的每一个节点都有父节点；树中的每一个节点都有0~N个子节点。例如节点4的父节点是节点2，子节点有2个，节点8和9。

树中的节点分为内部节点和外部节点（叶节点）。
内部节点：至少有一个子节点的节点。节点1~7都是内部节点。
外部节点/叶节点：没有子节点的节点。节点8~15都是外部节点。

一个节点的祖先节点包括它的父节点、祖父节点、曾祖父节点等等。例如节点8的祖先节点有节点1、4。
一个节点的后代节点包括它的子节点、孙子节点、曾孙节点等等。例如节点2的后代节点有节点4、5、8、9、10、11。
一个节点和它的后代节点构成了一个子树。例如节点4、8、9构成了一个子树。

节点的深度：从根节点到该节点的唯一路径的长。例如根节点（节点1）的深度是0，节点8的深度是3。
节点的高度：从该节点到它的叶节点的最长路径的长。例如根节点（节点1）的高度是3,节点8的高度是0。
树的深度：等于该树最深的叶节点的深度。该树的深度为3。
树的高度：等于该树根节点的高度。该树的高度为3。
实际上，不同人对深度和高度的定义有所差异。

二叉搜索树

二叉树（BinaryTree）：二叉树中的节点最多只能有两个子节点，左侧子节点和右侧子节点。
二叉搜索树（BinarySearchTree, BST）：是一种特殊的二叉树，但要求节点的左子节点存储的值比该节点的值要小，节点的右子节点存储的值比该节点的值要大。例子如下图所示。

二叉搜索树的遍历

二叉树的遍历有三种方式：中序遍历、先序遍历和后序遍历。

三种遍历方式的区别在于遍历节点顺序的不同（就一个节点和它的左子节点、右子节点而言）。
中序遍历：左子节点、节点、右子节点
先序遍历：节点、左子节点、右子节点
后序遍历：左子节点、右子节点、节点

在二叉搜索树中，中序遍历也就是以从最小到最大的顺序访问所有节点。中序遍历的一个应用就是对树进行排序。

二叉搜索树的搜索和移除

对于二叉搜索树的搜索，有三种搜索类型：最小值、最大值和搜索特定的值。

在二叉搜索树中移除一个节点，有三种情况：

  • 移除一个叶节点
    将该节点赋予null值，并且通过返回null将对应的父节点指针赋予null值。
  • 移除有一个左侧或右侧子节点的节点
    跳过这个节点，直接将父节点的指针指向这个节点的子节点。
  • 移除有两个子节点的节点
    找到该节点右侧子树中最小的节点，并且用这个最小的节点的值去更新该节点的值，同时把最小的节点移除，最后向该节点的父节点返回更新后节点的引用。

关于二叉树

二叉搜索树存在一个问题：树的一条分支有很多层，其他分支只有几层，也就是说非常不平衡，会引起一些性能问题。解决这一问题可以使用平衡二叉树满即AVL树。AVL树任何一个节点左右两侧子树的高度差不会大于1。

红黑树是一种特殊的AVL树，性能优于AVL树。
