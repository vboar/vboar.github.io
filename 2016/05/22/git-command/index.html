修改和提交

首先需要了解一下Git中的工作区和暂存区。

图中的index即暂存区（stage, index），暂存区实际上是一个包含文件索引的目录树，文件实际上是保存在Git对象库（.git/objects文件夹中，即图中的objects）中，文件索引建立了文件和对象库中对象实体之间的对应关系。
master即master分支所代表的目录树，HEAD实际上是指向master分支的一个游标。
工作区为除.git目录外的所有目录和文件，工作区是我们当前真实可见的。

Git中文件分为未跟踪（Untracked）和已跟踪，其中已跟踪文件的状态分为未修改（Unmodified）、已修改（Modified）和已暂存（Staged）。

撤销

如果要回退到之前的版本，可以使用下述命令：

git reset --hard和git reset --soft的区别在于：
git reset --soft执行时，在这条commit之后的修改全部放进暂存区里，而git reset --hard则丢弃这些修改。

git reset和git revert的区别在于：
git reset会回到之前的某个版本（同一个ID），HEAD向后移动。
git revert则是生成一个新的提交，HEAD向前移动。

分支衍合（rebase）和分支合并（merge）的区别在于：
rebase会把当前分支移动到指定分支的后面，有效地把所有指定分支上新的提交并入过来。但是，rebase为指定分支上每一个提交创建一个新的提交，重写了项目历史，并且不会带来合并提交。
